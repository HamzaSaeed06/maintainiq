# Production UI/UX Rebuild Prompt

Your task is to completely rebuild the entire frontend UI/UX from scratch while keeping the existing functionality intact.

Do not simply redesign screens. Think and work like a Senior Product Designer + Senior Frontend Engineer building a production-ready SaaS product used by thousands of users.

## Core Objective

Create a premium, modern, responsive, highly polished, production-quality user interface that feels like software from companies such as Stripe, Linear, Vercel, Notion, Figma, Framer, or Raycast.

Every page, every component, every interaction, and every pixel should look intentional.

---

# Think Beyond the Design

Do not only improve what already exists.

Review the entire project and independently identify opportunities to improve:

* Layout
* User Experience
* Visual hierarchy
* Accessibility
* Navigation
* Empty states
* Loading states
* Error handling
* Forms
* Tables
* Cards
* Buttons
* Modals
* Mobile experience
* Desktop experience
* User flow

If something feels outdated or unprofessional, redesign it without asking for permission.

---

# Production Quality Expectations

The application should never feel like:

* a student project
* a template
* a UI kit demo
* an unfinished MVP

Instead it should feel like a real commercial product.

Every page should have:

* proper spacing
* balanced layouts
* consistent alignment
* excellent typography
* clean visual hierarchy
* intuitive navigation
* polished interactions
* professional responsiveness

---

# Visual Design Principles

Maintain a consistent design language across the entire application.

Ensure:

* consistent border radius
* consistent shadows
* consistent spacing system
* consistent icon sizing
* consistent typography scale
* consistent color palette
* consistent component behavior
* consistent hover styles
* consistent transitions

Avoid visual inconsistency.

---

# Modern UX

Redesign the experience with modern UX principles.

Improve:

* user flow
* discoverability
* readability
* navigation
* interaction feedback
* information architecture
* visual hierarchy

Reduce unnecessary clicks wherever possible.

Users should always know:

* where they are
* what they can do
* what happens next

---

# Responsive Design

Every screen must work perfectly on:

* Mobile
* Tablet
* Laptop
* Desktop
* Large screens

Nothing should overflow.

Nothing should break.

No horizontal scrolling.

Layouts should adapt naturally.

---

# Micro Interactions

Every interactive element should feel alive.

Examples include:

* hover effects
* button press feedback
* subtle scaling
* elevation changes
* icon animations
* ripple effects where appropriate
* smooth transitions
* animated focus states
* animated dropdowns
* animated accordions
* animated tabs
* animated navigation
* animated cards

Animations should enhance usability—not distract.

* smooth modal animations

# Independent Decision Making

Do not ask for permission for every UI improvement.

Review the entire project carefully.

If you identify opportunities to improve the interface, UX, responsiveness, animations, accessibility, or component quality, implement those improvements proactively while preserving functionality.

Think like a product team preparing this application for a real production launch.

The final result should feel premium, polished, modern, scalable, intuitive, and indistinguishable from a professionally designed commercial web application.
